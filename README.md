# Karatsuba法と単純計算の比較

## 実行環境

Python 3.11.9 

## 概要
カラツバ法と呼ばれる掛け算の計算手法と通常の筆算型の掛け算アルゴリズムをPythonで実装し、実行時間を比較するプログラムを作成しました。

単純計算は、各桁同士をすべて掛け合わせるため、桁数をnとすると時間計算量はO(n^2)となります。
一方、カラツバ法は分割統治法を用いて掛け算の回数を減らすことで、計算量をO(n^{log₂3}) ≒ O(n^{1.58}) に抑えます。

## 制作理由

カラツバ法について調べていく中で、分割統治法と再帰処理により、時間計算量の解析をする際にシェルピンスキーのギャスケットと呼ばれるフラクタル図形が出現することを知りました。
一見すると無関係である「掛け算の高速化」と「フラクタル構造」が再帰処理という共通点でつながっている点に興味を持ち、実際に実装し、比較してみようと考えました。

## 実装内容

- 桁ごとの単純計算による掛け算
- 再帰を用いたKaratsuba法
- 実行時間の計測
- 計算結果がPython標準の掛け算結果と一致するかの確認

## 実行結果例

本プログラムでは、大きな整数同士の掛け算を想定し、以下のような数値を用いています。

123456789^100

987654321^100

(約900桁)

単純計算: 1.725242秒

カラツバ法: 0.032930秒

結果:約52倍の高速化

## 補足

単純計算のO(n^2)とカラツバ法のO(n^1.58)では、比率n^2/n^1.58=n^0.42より900^0.42≒17より、約17倍差になりますが、今回の結果は、約52倍差となりました。これの具体的な原因として、本プログラムでは、単純比較ではforループを用いて実装、カラツバ法では再帰処理のベースケースでPython標準の乗算を使用していることが原因であると推測できます。
そのため、純粋な理論計算量だけでなく、Pythonの実行効率の影響も含まれています。
